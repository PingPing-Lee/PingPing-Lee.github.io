之前项目中有遇到过一个 hybrid 项目，简单点理解就是我们的 H5页面嵌在运营商他们自己 APP 的 WebView 里面，主要是解决 RN 满足不了的需求。

H5的项目相对较 RN 来说，调试会稍微麻烦点～～

# 环境确认

目前只找到安卓机调试的方法，所以在开发之前，需要先具备一部安卓手机
同时确认运行环境满足以下要求：

  • Chrome 版本 >= 32
  • Android 版本 4.0 +

确定了版本 OK，那就安装相应的 APP 吧，一定要确保该 APP 的安装包是已经打开 WebView 调试权限的，并请确认调试环境是预发还是线上

# 开启开发者模式

  • 使用 USB 数据线将手机与电脑相连，记住一定要使用原装数据线，不要使用山寨的数据线或一线多头的数据线。
  • 找到安卓机的开发者选项并打开，下滑找到并打开 USB 调试，并允许调试
  • 不同设备进入开发者模式的方式有一点点不同，一般查找路径是：设置 --> 更多设置 --> 开发者选项 --> usb调试，找不到的请自行摸索...

# 打开浏览器

在Chrome 浏览器地址栏输入：chrome://inspect/#devices 并勾选 Discover USB devices 选项
如果已成功打开 USB 调试，就能看到当前手机型号，如下：

# 打开 APP

打开刚刚下载的 APP 并登录，点击入口进入需要调试H5面板界面，这个时候你就会看到浏览器有个WebView的URL路径，点击 inspect，就可以进入调试的界面：

如果到了这一步，WebView页面没出来，而这些流程你都确保OK，那试试看：

  • 多次点击 Discover USB devices , 看看有没有出来
  • 使用原装数据线，不要使用山寨的数据线或一线多头的数据线
  • 重新插拔 USB 数据线，让手机处于传输文件状态
  • 关闭电脑相关的应用助手
  • 重启大法好，先手机，不行再电脑
  • 换台手机，或者确保，你的app是开启了 WebView 调试
  • Chrome Inspect 查看页面时，Chrome 需要从 https://chrome-devtools-frontend.appspot.com 加载资源，如果你得到的调试界面是一片空白，你可能需要翻墙。

# 进入调试界面

完成了上面的步骤，就已经把APP内 WebView 中的内容在Chrome中打开了，接下去我们要做的是把这个 WebView 中的内容替换成本地代码的调试面板
