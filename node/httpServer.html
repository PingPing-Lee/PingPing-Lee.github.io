# HTTP

了解 HTTP 服务之前，可以先移步这里看一下 HTTP 是什么

简单的理解就是 HTTP 把网页打包成网络数据包，在目的地又拆成网页
以一个网页请求为例，包含以下两次 HTTP 包交换：

  • 浏览器向 HTTP 服务器发送请求 HTTP 包
  • HTTP 服务器向浏览器返回 HTTP 包

可以看出来一次网络请求就是浏览器和服务端通信的过程，那么在这个过程中服务器做了什么呢？

  • 服务器上有一个 HTTP 服务程序，程序首先把浏览器传送的 HTTP 请求报文解析出来
  • 再把返回内容通过 HTTP 协议打包，返回给浏览器端，浏览器再根据 HTTP 协议解析返回的报文，形成网页

若要使用 HTTP 服务器和客户端，则可以 require('http')，这里简单的建立一个 HTTP 的请求：

// fs 文件模块
const fs = require('fs');
// require http 内置模块
var http = require('http');

// 创建一个能够进行http服务的程序实例
http.createServer(function (request, response) {
    // request 请求头中包含 url, method, headers 等信息
    if (request.url == '/favicon.ico') {
        response.writeHead(200);
        response.end();
        return;
    }
    response.writeHead(200, {'Content-Type': 'text/plain'});
    // 这里需新建文件 1.html，并读取文件流
    fs.createReadStream(__dirname + '/1.html').pipe(response)
}).listen(3000); // listen 监听 3000 端口

在 NodeJs 中 http 服务的开发领域里，比较权威的 HTTP 服务框架就是 Express

# Express

核心功能：

  • 路由
    • 请求包传送到服务器，服务器会根据包的内容进行分发到不同的逻辑单元处理，分发的过程就叫做路由
  • request/response 简化
    • request: pathname、query 等
    • response: send()、json()、jsonp() 等
  • 中间件
    • 更好地组织流程代码
    • 异步会打破 Express 的洋葱模型

基础示例代码如下：

const express = require('express')
const app = express()

// 通过 app.get 设定 /favicon.ico 路径的路由
// .get 代表请求 method 是 get，所以这里可以用 post、delete 等。这个能力很适合用于创建 rest 服务
app.get('/favicon.ico', function(request,response) {
    // response.writeHead(200);
    // response.end();
    // 一句 status(200) 代替 writeHead(200); end();
    response.status(200);
    response.send();
    return;
})

// send接口会判断你传入的值的类型，文本的话则会处理为text/html
// Buffer的话则会处理为下载
app.get('/', function(request,response) {
    response.send(fs.readFileSync(__dirname+'/1.html', 'utf-8'))
    // response.writeHead(200);
    // fs.createReadStream(__dirname + '/1.html').pipe(response)
})

app.listen(3333, () => {
    console.log('listen start')
})

上述代码中我们可以看出，express 返回一个 app 类，且拥有 get，listen方法，根据以上功能我们来仿写一个简版 Express，示例代码如下：：

//  myExpress.js
const http = require('http')
const fs = require('fs')
const url = require('url')
const router = []
class Application {
    get(path, handler){
        router.push({
            path,
            method: 'GET',
            handler
        })
    }
    listen(){
        const server = http.createServer((req, res) => {
            const { pathname } = url.parse(req.url, true)
            router.find(route => pathname === route.path && req.method === route.method).handler(req, res)
        })
        server.listen(...arguments)
    }
}
module.exports = function (){
    return new Application()
}

但是因为 Express 对异步的支持不太好、洋葱模型的不完善，社区便有了 Koa，

# Koa

也是 nodejs 社区比较权威的 HTTP 服务框架，为了弥补 Express 的缺陷而诞生

  • 使用 async function 实现的中间件
    • 有“暂停执行”的能力
    • 在异步的情况下也符合洋葱模型
  • 比 Express 更极致的 request/response 简化
    • 把 Request 和 Response 都归纳到 ctx 中，使用时 ctx.request、ctx.response
    • ctx.status = 200
    • ctx.body = 'hello world'
  • 精简内核，不会绑定任何的中间件，如路由的处理，放到中间件里，让使用者通过中间件实现功能，并没有自带的路由中间件

基础示例代码如下：

// require http 内置模块
var http = require('http');
const fs = require('fs');
const koa = require('koa');
// 通过中间件 koa-mount 来处理路由
const mount = require('koa-mount');

const app = new koa();

app.use(
    mount('/favicon.ico', function (ctx) {
        // koa比express做了更极致的response处理函数
        // 因为koa使用异步函数作为中间件的实现方式
        // 所以koa可以在等待所有中间件执行完毕之后再统一处理返回值，因此可以用赋值运算符
        ctx.status = 200;
    })
)
app.use(
    mount('/', function(ctx) {
        ctx.body = fs.readFileSync(__dirname+'/1.html', 'utf-8');
        return;
    })
)
app.listen(3000)

Express 和 Koa的对比：

  • Express 门槛更低，Koa 更强大优雅
  • Express 封装更多东西，开发更快速，Koa 可定制型更高
  • Express 更适用与小型应用，Koa 更适用于大型可维护性的定制应用，适用场景不同，其实并没有优劣之分

上述所有示例代码中都引入了内置模块 fs，这里对 fs 简单介绍一下

# FS

fs（文件系统） 模块可用于与文件系统进行交互（以类似于标准 POSIX 函数的方式）。

要使用此模块：

const fs = require('fs')

所有的文件系统操作都具有同步的、回调的、以及基于 promise 的形式。

// 同步的示例
const data = fs.readFileSync('./package.json')
console.log(data.toString())

// 但是日常开发中更多的是异步回调的操作
const fs = require('fs')
fs.readFile('./package.json', (err,data) => {
    console.log(data.toString())
})

// 其实我们可以使用内置的 promisify 进行异步操作，如下：
const fs = require('fs')
const { promisify } = require("util")
const readFile = promisify(fs.readFile)
readFile('./package.json').then(data => {
    console.log(data.toString())
})
// 可以改写成如下写法
(async () => {
    const fs = require('fs')
    const { promisify } = require("util")
    const readFile = promisify(fs.readFile)
    const data = await readFile('./package.json')
    console.log(data.toString())
})()
