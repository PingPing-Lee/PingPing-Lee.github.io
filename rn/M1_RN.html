# M1芯片的MAC机上搭建RN开发环境

# 安装 Xcode

进入 App Store 中安装 Xcode，安装好之后再进入后续步骤。
这里需要注意 xcode 和 Mac电脑的系统版本有着一定的对应关系，请戳这里 (opens new window)

# 安装 Homebrew

Homebrew ，并使用它来管理 Mac 中的各类环境依赖
根据官方规划，ARM 版 Homebrew 必须安装在 /opt/homebrew 路径下，而非此前的 /usr/local/Homebrew。
由于官方的安装脚本还未更新，所以打开终端，通过如下命令手动安装：

  1. 切换到 /opt 目录

    cd /opt

  2. 创建 homebrew 目录

    mkdir homebrew

  3. 指令

    curl -L https://github.com/Homebrew/brew/tarball/master | tar xz --strip 1 -C homebrew
<!-- 如果报错请试试下面这个： -->
/bin/zsh -c "$(curl -fsSL https://gitee.com/cunkai/HomebrewCN/raw/master/Homebrew.sh)"

    提示：
    如果安装和使用过程中报错，可能是因为当前用户对 /opt/homebrew 路径没有操作权限。
    对此，可以通过 sudo chown -R $(whoami) /opt/homebrew 更改该目录权限

  4. 配置 brew 的环境变量
    默认情况下，ARM 版 Homebrew 用来安装程序的路径 /opt/homebrew/bin 并不在环境变量 PATH 中，因此终端无法检索到该路径下的 brew 程序，需要配置 .zshrc 文件，找到 .zshrc 文件到方法如下：

    • 方法一：
      • 快捷键 Command+Shift+. 可以显示隐藏文件、文件夹，再按一次，恢复隐藏；
    • 方法二：
      • 在终端输入 defaults write com.apple.finder AppleShowAllFiles -boolean true ; killall Finder，然后回车

    找到 .zshrc 文件之后，复制以下内容到 文件中：

    path=('/opt/homebrew/bin' $path)
export PATH

  5. brew -v 查看
    然后重新启动终端，查看 brew是否安装好：

# 安装 Nvm

Nvm 是一个管理 Node 版本的工具，当项目多了以后，总会碰到需要切换 Node 环境的时候，所以推荐使用 Nvm 来安装 Node。

  1. Homebrew 安装 Nvm

    若是你已经按上述步骤安装好了 Homebrew，那么你可以直接通过 Homebrew 的命令轻松安装 Nvm，打开终端，在根目录中输入：

    brew install nvm

    安装完成后brew会给出一些提示，如下图：

  2. 设置 Nvm 的环境变量 根据上图提示

    • 在终端创建 .nvm 目录
    mkdir ~/.nvm
    • 编辑 .zshrc 配置文件，如果没有请创建，然后把上图红色框中内容复制到文件中
    vi ~/.zshrc
    • 在 ~/.zshrc 配置文件内容最后添加上图红色框中的内容
    export NVM_DIR="$HOME/.nvm"
[ -s "/usr/local/opt/nvm/nvm.sh" ] && \. "/usr/local/opt/nvm/nvm.sh"  # This loads nvm
[ -s "/usr/local/opt/nvm/etc/bash_completion.d/nvm" ] && \. "/usr/local/opt/nvm/etc/bash_completion.d/nvm"  # This loads nvm bash_completion
    • :wq 保存并退出, 使用 source 命令使配置生效
    source ~/.zshrc

  3. 查看

    • 查看配置是否生效：
    • nvm -v 查看版本
    • nvm -help
    nvm -help
...
Note:
to remove, delete, or uninstall nvm - just remove the `$NVM_DIR` folder (usually `~/.nvm`)
    • 在终端执行 nvm -help 命令，我们可以看到的最后一句话，删除卸载nvm只需要移除$NVM_DIR文件夹，一般是~/.nvm
    • 注意：nvm安装流程大同小异，主要是最后的配置阶段里面的路径一定要配置正确，这样才能确保你的nvm命令可用。
    • 重启终端即可使用 NVM

  4. 常用 nvm 命令

  • 查看已安装的版本：nvm ls
  • 查看可以安装的版本：nvm ls-remote
  • 安装指定的版本：nvm install version
  • 删除指定的版本：nvm uninstall version
  • 使用选定的版本：nvm use version
  • 安装最新的可用版本 nvm install node
  • 安装最新的 LTS 版本 nvm install --lts
  • 设置默认 node 版本 nvm alias default version
  • 解除当前版本绑定 nvm deactivate version

# 安装 Node

接下来我们通过 nvm 来安装 Node，打开终端输入以下指令： nvm install stable # 安装最新稳定版 然后查看 Node 是否安装好：node -v

# 如何安装 v14 及以下的老版本 Node

此时通过 nvm 安装最新的 node 版本没有什么问题，可以正常安装，这是因为最新版本的nvm和node，已经对M1做了兼容处理。 但是当通过 nvm 安装 v14 及以下的老版本 Node 时，就会报错，究其原因还是因为低版本的 node 并不是基于 arm64 架构的，所以不适配 M1 芯片。在这里教大家两个方法，就能成功安装上低版本 Node。

  1. 方法一： 在终端输入以下命令：
    • arch -x86_64 zsh 通过这个命令可以让 shell 运行在Rosetta2下 之后你可以通过 nvm install v14 来安装低版本 Node 在此之后，您可以不用在 Rosetta2 中就可以使用安装的可执行文件，也就是说，您可以将 Node v15与其他节点版本互换使用
  2. 方法二：通过 Rosetta2 来启动终端，这样通过 Rosetta2 转译到 x86 架构中执行安装，也一样可以安装成功
    • 在 finder 中，点击应用程序，并在实用工具中找到终端 (Terminal)
    • 右键终端，点击显示简介(Get Info)
    • 勾选 使用 Rosetta 打开
    • 重启终端，并执行 nvm install v14 来安装低版本 Node

至此 M1芯片的MAC机上就已经完成了React Native开发环境的前期搭建，完成这部分工作后，就可以搭建React Native环境了，具体的安装步骤请参考 RN 中文网
