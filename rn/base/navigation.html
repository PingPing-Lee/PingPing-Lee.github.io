虽说 React Native 框架并未将导航内置，但作为用来管理页面之间的链接的工具，导航又是日常开发中必不可少的

目前官方推荐的、主流的导航是 React Navigation，注意不是 React Native Navigation～

# 创建导航

React Navigation 导航分了好几个子库，下面我们就来看一下创建导航的示例代码：

// 从 @react-navigation/native 库中引入 NavigationContainer 容器组件
import { NavigationContainer } from '@react-navigation/native';
// 从 @react-navigation/native-stack 库中引入 createNativeStackNavigator 方法
import { createNativeStackNavigator } from '@react-navigation/native-stack';

// 创建的Stack 是原生堆栈导航，用来创建页面和收集页面的
const Stack = createNativeStackNavigator();

function App() {
  return (
    // 用容器组件包裹住整个 App 的 JSX 元素
    <NavigationContainer>
        {/* Stack.Navigator 用来收集该导航下有哪些页面 */}
      <Stack.Navigator initialRouteName="ListItem">
        {/* Stack.Screen 用来创建页面 */}
        {/* name 和 component 命名也可以保持一致 */}
        <Stack.Screen name="listItem" component={ListItem} />
        <Stack.Screen name="itemDetail" component={ItemDetail} />
        ...
      </Stack.Navigator>
    </NavigationContainer>
  );
}

上述代码中只是使用 Stack.Screen 创建了列表项页面 ListItem 和 列表项详情页面 ItemDetail，但是想要实现页面直接带参数跳转要如何写呢？

# 带参跳转

React Navigation 给出的跳转方法是 navigation.navigate 函数，其中的 navigation 对象是当函数组件通过 Stack.Screen 生成页面时，默认携带的对象

具体的示例代码如下：

function ListItem({navigation}) {
  return (
    ...
        <Pressable
            onPress={()=> {
                {/* 
                 * 第一个参数是要跳转的页面名字
                 * 第二个参数是跳转时要携带的自定义参数 
                */}
                navigation.navigate('itemDetail', {
                    describe: '列表项详情页面，并携带参数跳转',
                });
            }}
        >
            <Text>我是列表项</Text>
        </Pressable>
    ...
  );
}

# 解析参数

跳转携带的参数在跳转之后的页面中，会被挂载在 route 的 params 属性上，这里的 route 对象和 navigation 对象一样，是函数组件使用 Stack.Screen 创建页面时携带的对象

function ItemDetail({route}) {
    // 获取跳转携带的参数并解析
    const { describe }  = route.params
    return (
    <View>
        <Text>{describe}</Text>
    </View>
    )
}

总结下来，日常开发中通过创建导航、携带参数跳转页面、接收页面解析参数这三步，就能实现最基础的页面跳转了，但是你可能会想到要如何实现从 ItemDetail 页面返回到 ListItem 页面呢？

其实 React Navigation 已经处理好了，它会给页面创建导航栏，并帮助页面处理返回相关手势动画

但是想要更好地使用 React Navigation ，就要理解它的两个配置项和导航路由对象

# 配置项

使用 React Navigation 创建的页面，属性 params 和 options 比较常用到：

  • params：挂在 route 上的对象，是开发者自定义参数，通常用来渲染页面主体的数据
  • options：是导航相关的配置属性，包括手机顶部的状态栏 status、页面的标题栏 header、导航相关手势 gesture 等等

在一些场景下，我们可以使用 setParams 和 setOptions 方法来重新自定义 params 和 options

# params

具体实现方式的示例代码如下：

// 页面声明
<Stack.Screen name="itemDetail" initialParams={{ describe1: '携带参数的默认值1', describe2: '携带参数的默认值2' }} component={ItemDetail} />


// ItemDetail
function itemDetail({route, navigation}) {  
    const { describe } = route.params 
    return (
        <>
            <Text>{describe1}</Text>
            <Text>{describe2}</Text>
            <Pressable
                onPress={() =>{
                    navigation.setParams({ describe1: '重新设置之后的参数1'})  
                }}
            >{describe}</Pressable>
        </>
    )
}

点击 Pressable 按钮之后，会发现页面上的 describe1 进行更新，但是 describe2 并未更新，所以 navigation.setParams 重置自定义 params 参数时，会将新旧 params 对象进行合并，并使用合并后的 params 重新渲染页面

# options

options 具体的配置非常多，这里只列举一些常用的配置项，大致分成了 3 类：

# header 类

  • title：是字符串，用于设置导航标题
  • headerBackTitleVisible：是布尔值，用于决定返回按钮是否显示回退页面的名字。默认是 true 显示（iOS 专属）
  • headerShown：是布尔值，用于决定是否隐藏导航头部标题栏
  • header：接收一个返回 React 元素的函数作为参数，返回的 React 元素就是新的导航标题栏

# status 类

控制屏幕顶部状态栏用的，也可使用 React Native 框架提供的组件进行代替

  • statusBarHidden：是布尔值，决定了屏幕顶部状态栏是否隐藏

# 手势动画类

  • gestureEnabled：是布尔值，决定了是否可用侧滑手势关闭当前页面（iOS 专属）
  • fullScreenGestureEnabled：是布尔值，决定了是否使用全屏滑动手势关闭当前页面（iOS 专属）
  • animation：是字符串枚举值，控制了打开或关闭 Stack 页面的动画形式，默认“default”是页面从右到左地推入动画，也可以设置成其他类型的动画，比如“slide_from_bottom”是页面从下到上的推入动画和从上到下的推出动画
  • presentation：是字符串枚举值，控制了页面的展现形式，其主要作用是设置页面弹窗。常用的配置值是 “transparentModal” 它会将页面展示为一个透明弹窗

具体用法的示例代码如下：

<Stack.Screen
    name="itemDetail"
    component={ItemDetail}
    options={{
        headerShown: false,
        fullScreenGestureEnabled: true,
        ...
    }}
/>

以上是通过配置设置 options 参数的过程，那如何通过 setOptions 方法在当前页面修改 options 配置呢？示例代码如下：

function ItemDetail({ navigation}) {
  // React.useEffect() 异步副作用回调，执行 setOptions 会导致闪屏，不推荐使用。

  // 页面初始化时，为了页面不抖动, 使用同步的方法渲染页面
  React.useLayoutEffect(() => {
    navigation.setOptions({
      headerShown: false,
      fullScreenGestureEnabled: true,
    });
  }, [navigation])

  // 点击按钮后，异步设置
  const handlePress = () => {
    navigation.setOptions({
      title: '新标题',
    });
  }

  return (
    <Text onPress={handlePress}>设置新标题</Text>
  );
}
