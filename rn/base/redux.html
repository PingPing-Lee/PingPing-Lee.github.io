在日常的开发中，通常使用 Redux 来管理应用状态，如果觉得 Redux 模板代码多，也可以使用 Redux Toolkit 来解决一部分

# Redux

对于父子组件通信直接通过 props 传递就可以，但是对于相邻组件或者潜逃好多层的组件之间的通信，如果还是通过逐层传递的方式，虽然可以解决跨组件更新需求，但是却带来了性能损耗和维护成本高等问题。

因此在实际项目开发中，当管理状态的逻辑特别复杂时，当有大量的全局状态需要管理时，当应用状态频繁更新遇到性能瓶颈时，当多人协作开发需要统一风格时，就可以考虑使用 Redux 了

# 工作原理

一句话概括 Redux 的核心原理就是：State 驱动 View 更新，用户操作 View 触发 Action ，再通过 Action 来更新 State

我们都知道这里提到的 View 是 JSX 视图，那其他俩个呢？

State 是一个存储在全局中的对象，用于描述整个应用的状态，State 对象中包含的是全局状态，全局状态可以有一个也可以有多个，状态的值既可以是原始数据类型 number、string 等，也可以是更复杂的 object、array 数据类型

Action 是一个包含 type 字段的对象，用来描述发生了什么事情，具体格式如下：

{ type: 'ACTION_TYPE', params: '...' }

上述代码中的 params 字段，是可以自己定义的。在有一些工具，会统一将其定义为 payload 字段，省去了字段名不确定的麻烦事

那当 Redux 收到 Action 通知后，如何更新全局状态呢？这就要用到 Reducer 了。

# Reducer

Reducer 就是一个普通的状态更新函数。Redux 会将当前应用的状态 State 和指令 Action 作为参数，传给 Reducer 函数，并接收一个 Reducer 返回的 newState 作为新全局状态。函数示意如下：

const reducer = (state, action ) => newState

在一个简单的计数应用中，Reducer 可以这样写：

// increment 函数就是处理分片状态的 Reducer 函数
function increment(state = 0, action) {
    return state+1
}

// counterApp 是 Redux 管理所有全局状态的函数
function counterApp(state = {}, action) {
  return {
    counter: increment(state.counter, action),
  }
}

上述代码中，counterApp 函数接收当前状态 state 和发出的指令 action 作为参数，并且将这些参数传给 increment，由 increment 更新函数来更新状态。

这端代码很简单，也只有一个全局状态的函数和一个处理分片状态的 Reducer 函数，但是在实际项目开发中或者复杂应用中，只用一个函数来处理所有状态是不现实的，所以 Redux 既有管理所有全局状态的函数，又有若干个能够处理不同分片状态的 Reducer 函数

总结下来就是，在初始化时，Redux 通过 Reducer 来初始化 State，State 驱动 View 渲染。在更新状态时，用户操作 View 触发 Action，Action 和当前 State 会被分发给处理分片状态的 Reducer 函数，由 Reducer 函数来执行更新逻辑和返回新的 State，并最终刷新 View，这就是 Redux 的核心原理

# Redux Toolkit

Redux Toolkit 是 Redux 社区和官方团队，经历多年的摸索出的一套最佳实践，不仅能够避免常见的 Redux 使用误区，还能少写很多代码

Redux Toolkit 的本质是提供了一些工具函数，简化纯手写 Redux 代码的冗余逻辑，其中最重要的两个工具函数是：

  • configureStore：管理所有全局状态的函数，它的返回值是一个 Store 对象
  • createSlice：管理分片全局状态的函数，其返回值是一个分片对象，该对象上最重要的两个属性是：
    • actions：创建分片 action 的函数集合
    • reducer：已经创建好的分片 reducer

# React Redux

在日常开发中管理状态时，还有一个常用的库，那就时 react-redux，提供了 Provider 组件和 useSelector，useDispatch 钩子函数：

  • Provider： 是一个组件，接收存储所有全局状态的 Store 对象作为参数。Provider 组件底层用的是 useContext，它为整个应用的其他组件提供获取 Store 对象的能力
  • useSelector：从 Store 中获取当前组件需要用到的状态
  • useDispatch：用于发送指令的钩子函数，其返回值是 dispatch 函数，而 dispatch 函数的入参是 action

推荐同时使用 Redux Toolkit 和 React Redux，实现全局状态管理，具体实现步骤请继续往下看

# 1. 使用 Provider 组件向其他组件提供获取 Store 的能力

一般而言，Provider 组件通常是最顶层的组件，它包裹住了整个应用。 示例代码如下：

import { Provider } from 'react-redux';
// store 的创建先省略，下一步再说
const store = ...

function App(){}

// 根组件
export default function Root() {
  return (
    <Provider store={store}>
      <App />
    </Provider>
  );
}

AppRegistry.registerComponent('App', () => Root);

上述代码中我们可以看到 Provider 包裹住整个应用代码，同时 Provider 接收了 store 作为参数，因此 App 组件及其所有的子组件都能从 store 中获取任意的全局状态

# 2. 创建 store

上面代码中并没有声明 store 的创建方式，其实创建 store 用到的函数是 configureStore

import { configureStore } from '@reduxjs/toolkit';

// 创建分片先省略，下一步继续
const counterSlice = ...

const store = configureStore({
  reducer: {
    counter: counterSlice.reducer,
  }
});

console.log(store.getState()) // { counter }

configureStore 函数的入参中最关键的对象属性是 reducer 属性， 上述代码中入参对象中的 reducer 属性是最关键的对象属性，reducer 的内容是有分片状态 counter 组成的，counter 又是通过分片 counterSlice 中的 reducer 属性来创建的

# 3. 创建分片

上面提到了分片，但是未涉及如何创建，其实创建分片用到的函数是 createSlice

import { createSlice } from '@reduxjs/toolkit';

const initialState = {
  value: 0,
};

const counterSlice = createSlice({
  name: 'counter',
  initialState,
  reducers: {
    increment: (state, action) => {
      // 可以直接修改的“状态”
      state.value += action.payload;
    },
  },
});

创建分片需要三个参数：

  • name：分片的名字，字符串类型
  • initialState：分片的初始化状态
  • reducers：对象类型，用于创建分片 action 和分片 reducer

createSlice 函数会自动生成分片 action creators 和分片 reducer， 并将这两个自动生成的值挂在其返回对象 counterSlice 的 actions 属性和 reducer 属性上

此时应用中的组件就能够使用分片 State 和分片 Action 来展示和改变 UI 视图了，具体该如何获取呢？

# 4. 如何使用

import { useSelector, useDispatch } from 'react-redux';

function App() {
  // 使用 useSelector 按需获取状态
  const counter = useSelector((state) => state.counter.value);
  // 使用 useDispatch 获取 dispatch 函数
  const dispatch = useDispatch();

  const handlePress = () => {
    // 调用 dispatch(action) 发送指令
    dispatch(counterSlice.actions.increment(10));
  };

  return (
      <View style={styles.box}>
        <Text>{counter}</Text>
        <Text onPress={handlePress}>+10</Text>
      </View>
  );
}

useSelector 的入参是一个函数，你可以通过这个函数从所有 state 中选择该组件中用到的 state

总结下来 整个过程大致分为以下步骤：

  • 使用 Provider 组件向应用的其他组件提供获取 Store 的能力
  • 使用 configureStore 函数创建 Store
  • 使用 createSlice 函数创建分片
  • 使用 useSelector 获取分片 State
  • 使用 useDispatch 生成的 dispatch 来发送 action
