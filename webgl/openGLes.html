# 基础

像很多其他语言一样，在编写着色器程序是，应注意以下两点：

  • 程序是大小写敏感的，如定义变量 a 和 A 就不是同一个变量
  • 每一个语句都应该以一个英文分号(;)结束

# 执行次序

在 WebGL 中程序不是从第一行开始执行的，而是从 main() 函数开始执行，该函数不能接收任何参数，且在着色器程序中必须有且仅有一个 main() 函数

void main() {} 函数前有 void 关键字表示函数不返回任何值，如果函数有返回值也必须在定义函数是明确指定返回值的类型, float main() { return f }

# 注释

在着色器器程序中，注释的格式和JS 中是相同的：

  • 单行注释： // 后直到换行处的左右字符为注释
  • 多行注释： /* 和 */ 之间的所有字符为注释

# 数据值类型

GLSL 支持两种数据值类型：

  • 数值类型：
    • 整型数，没有小数点(.)的值，如 1，2...
    • 浮点数，有小数点(.)的值，如 3.14
  • 布尔值类型: 包括 true 和 false 两个布尔常量

注意不支持字符串类型

# 变量

GLSL 变量的声明和 JS 相同，都需要满足以下条件：

  • 只能是数字字母和下划线
  • 不能是数字开头
  • 不能是关键字或保留字
  • 不能以 gl_、 webgl_、webgl 作为开头

# 强类型语言

GLSL 中如果 main() 函数有返回值，要指定返回值的类型，而且在进行赋值操作 (=) 的时候，等号左右两侧的数据类型必须要一样，所以被称为 强类型语言，在编程时要时刻注意变量的类型

GLSL 的基本类型：

类型 描述
float 单精度浮点数类型
int 整型数
bool 布尔值

int i = 8 是没有问题，但是float f = 8就会报错，不能直接将整型赋值给浮点型，需要将整型转换成浮点型之后再赋值 int i = 8; float f = float(i); 这样就没问题

# 赋值与构造

我们使用等号(=) 来对矢量和矩阵进行赋值操作，且等号左右两边的变量/值的类型必须一致，左右两边的元素个数也必须相同

vec4 position = 1.0;

上面的代码就会报错因为vec4类型变量有四个元素，通常使用与数据类型同名的内置构造函数来生成变量，上面的应该改成如下代码：

vec4 position = vec4(1.0, 2.0, 3.0, 4.0);

这里的 vec4() 就是矢量的构造函数，还可以如下定义：

// 将v3 设置为 (1.0, 2.0, 3.0)
vec3 v3 = vec3(1.0, 2.0, 3.0);
// 使用 v3 的前两个元素，将v2 设为 (1.0, 2.0)
vec2 v2 = vec2(v3);
// 将 v4 设为 (1.0, 1.0, 1.0, ！.0)
vec4 v4 = vec4(1.0);

矩阵构造函数的使用方式和矢量构造函数类似，但是存储在矩阵中的元素是按照列主序排列的

# 纹理取样器

将 GLSL ES 支持的一种内置类型称为取样器，必须通过该类型变量来访问纹理

// 声明二维纹理
uniform sampler2D uSampler;
// 立方体纹理
uniform samplerCube uSamplerCube;
  • 有两种基本的取样器类型，sampler2D 和 samplerCube
  • 取样器变量只能声明为 uniform 变量
  • 唯一能赋值给取样器变量的就是纹理单元编号，而且必须使用 gl.uniform1i() 进行赋值

# 分支与循环

着色器中的分支与循环和JS 中没什么不同

通过 if 语句 if-else 语句进行分支判断

  • if() {} 和 if (){} else{}
  • if () {} else if () {} else {}

通过 for 语句实现循环，跳出循环可以选择 continue、break 或 discard 语句，其中 continue 和 break 的用法和 JS 一致，需要注意的是 discard 语句，只能在片元着色器中使用，表示放弃当前片元直接处理下一个片元

# 存储限定字

在 GLSL ES 中修饰变量的限定词除了常用的 attribute、uniform 和 varying 外，还有 const 限定字，表示着色器中某个变量是常量

声明事需要写在类型之前，而且声明之后不能再去改变，类似声明如下：

const mat4 ide = mat4(1.0)

总结下来就是：

  • attribute 只能出现在顶点着色器中，只能声明为全局变量，表示逐顶点信息。单个顶点的信息
  • uniform 可同时出现在 顶点着色器 和 片元着⾊器中，只读类型，强调一致性，用来存储的是影响所有顶点的数据。如变换矩阵
  • varying 从顶点着色器向片元着色器传递数据
  • const 声明一个常量，定义之后不能被改变

# 精度限定字

GLSL ES 引入了新的精度限定字，目的是为帮助着色器程序提高运行效率，削减内存开支

支持三种精度，其限定字分别为 lowp、mediump 和 highp，如下图：

示例代码如下：

mediump float f; // 中精度的浮点型变量
highp vec4 position; // 高精度浮点型元素的 vec4 对象

上述方式为每个变量都声明精度很繁琐的，也可以使用关键字 precision 来声明着色器的默认精度，且必须在顶点着色器或者片元着色器的顶部声明

precision 精度限定字 类型名称;
// 如下
precision mediump float; // 所有浮点数默认为中精度
