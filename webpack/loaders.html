# loader

webpack 是模块打包⼯具，⽽模块不仅仅是 JS，还可以是 CSS、图⽚或者其他格式

但是 webpack 开箱即用只支持 JS 和 JSON 两种文件类型，对于其它类型模块的支持需要通过 Loaders 去把它们转化成有效的模块，并且可以添加到依赖图中

Loaders 本身是一个函数，接受源文件作为参数，返回转换的结果，可以协助 webpack 打包处理待定的文件模块，目前比较常见的 Loaders 有如下表格：

名称 描述
babel-loader 转换ES6、ES7等JS新特性语法
css-loader 支持.css文件等加载和解析
less-loader 将less文件转换成css
file-loader 进行图片、字体等的打包
url-loader 处理css 中与 url 路径相关文件的打包
postcss-loader autoprefixer ⾃动补⻬ CSS3 前缀
ts-loader 将TS文件转换成JS
raw-loader 将文件以字符串的形式导入
thread-loader 多进程打包JS和CSS

loader 的调用过程：

熟悉了 loader 的调用过程之后，我们就来看看具体的 使用规则：

// 所有第三方文件模块的匹配规则
module: {
    rules: [
        {
            test: /.../, //  表示匹配文件类型，指定匹配规则
            use: ['...','...'], // 指定使用的 loader 名称
            // use 数组中指定的 loader 顺序是固定的，如果指定一个 loader 可以是对象或者字符串，两个及以上就需要用数组表示，且调用顺序是从后往前调用的
        }
    ]
}

下面来具体的看看常见 Loaders 的使用方法：

# babel-loader

假如你在项目的 JS 文件中输入如下代码，然后运行项目，

class Person {
    static info = 'aaa'
}
console.log(Person.info)

此时你会发现浏览器报错，这是因为 webpack 默认打包处理不了这种高级的 JS 语法，因此需要借助 babel-loader 来转换

babel-loader 是依赖 babel 的，因此先来看一下 babel 是什么

Babel 是一个 JavaScript 编译器
Babel 是一个工具链，主要用于将 ECMAScript 2015+ 版本的代码转换为向后兼容的 JavaScript 语法，以便能够运行在当前和旧版本的浏览器或其他环境中。 下面列出的是 Babel 能为你做的事情：

  • 语法转换
  • 通过 Polyfill 方式在目标环境中添加缺失的特性 (通过 @babel/polyfill 模块)
  • 源码转换 (codemods)
  • ...

我们需要配置 babel-loader，来解析并转换这些高级的 JS 语法，具体操作如下：

  1. 在终端中运行如下命令，安装 babel-loader 和相关插件的包
npm install babel-loader @babel/core @babel/preset-env -D
  1. 在根目录创建 babel 的配置文件，命名为 .babelrc，增加 ES6 的 babel presets 的相关配置内容如下：
module.exports = {
    presets: [ '@babel/preset-env' ] ,
}
  1. 打开 webpack.config.js文件， 配置 module.rules，修改内容如下：
// exclude 为排除项，表示 babel-loader 不需要处理 node_modules 中的 js 文件
module: {
    rules: [
        { test: /\.js$/ , use:'babel-loader',exclude: /node_modules/}
    ]
}

# css-loader

当我们在项目引入 css 文件时，会发现项目报错，提示我们打包失败，需要安装合适的加载器解决问题

这是因为在 webpack 中想要解析 css，首先需要使用 css-loader 来加载 .css 文件，并且转换成 commonjs 对象插入 js 代码中，再通过 style-loader 将 css-loader ⽣成的内容通过

标签挂载到⻚⾯的 head 部分，具体操作如下：

  1. 在终端中运行如下命令，安装处理 css 文件的加载器
npm install style-loader css-loader -D
  1. 打开 webpack.config.js，配置 module.rules 节点，新增 css 文件类型，具体内容如下：
module: {
    rules: [
+        {
+            test: /\.css$/ ,
+            use: ['style-loader','css-loader']
+        }
   ]
}

# less-loader

除了上面说到的 css-loader 和 style-loader，我们在日常的开发过程中可能还需要用到 less 或者 sass 样式，其实使用方法和上面类似：

  1. 首先安装处理样式文件的加载器
// 处理 less 样式
npm install less-loader less -D
// 处理 sass 样式，
npm install sass-loader node-sass -D
  • 这里需要注意 sass-loader 依赖 node-sass 模块把 sass 语法转换成 css，所以也需要安装 node-sass
  1. 安装完成之后，同样是配置 webpack.config.js 中的 module -> rules，新增规则如下：
module: {
    rules: [
+        {
+            test: /\.less$/ ,
+            use: ['style-loader','css-loader','less-loader']
+        },
+        {
+            test: /\.scss$/ ,
+            use: ['style-loader','css-loader','sass-loader']
+        }
    ]
}

# file-loader

在 webpack 中对svg、txt、excel、图片资源、字体文件等静态资源模块的处理，需要用到 file-loader

file-loader 原理是把打包⼊⼝中识别出的资源模块，移动到输出⽬录，并且返回⼀个地址名称

  1. 在终端中运行如下命令，安装相关的加载器
npm install file-loader -D
  1. 在 webpack.config.js 中，修改 module -> rules，新增规则如下：
module: {
    rules: [
        {
            test: /\.(png|jpe?g|gif|bmp|ttf|eot|svg|woff|woff2$)$/,
            use: {
                loader: "file-loader",
                // options额外的配置，⽐如资源名称
                options: {
                    // [name]打包前模块的名称 [hash]内容的哈希值 [ext]打包前模块的后缀
                    name: "[name]_[hash].[ext]",
                    //打包后的存放位置
                    outputPath: "images/"
                }
            }
        }
    ]
}

# url-loader

url-loader 可以处理 file-loader 所有的事情，但是遇到 jpg 格式的模块，会把该图⽚转换成 base64 格式字符串，并打包到 js ⾥，所以对⼩体积的图⽚⽐较合适，⼤图⽚不合适

url-loader 可以限定模块的体积，根据体积判断是否需要转换成 base64（文件大小低于指定的限制时），减少 http 请求

  1. 在终端中运行如下命令，安装相关的加载器
npm install url-loader -D
  1. 打开 webpack.config.js，修改 module -> rules，新增规则如下：
module: {
    rules: [
        {
            test: /\.(png|jpe?g|gif)$/,
            use: {
                loader: "url-loader",
                options: {
                    name: "[name]_[hash].[ext]",
                    outputPath: "images/",
                    //⼩于2048，才转换成base64
                    limit: 2048
                }
            }
        }
    ]
}

# postcss-loader

在设置 CSS 样式时，有很多时候需要我们考虑到兼容性问题，不然设置了也没效果，我们可以通过配置 postCSS 来实现自动添加 CSS 的兼容前缀

  1. 在终端中运行如下命令，安装包
npm install postcss-loader autoprefixer -D
  1. 在根目录创建 postcss 的配置文件，命名为 postcss.config.js ，配置内容如下：
module.exports = {
    plugins: [ 
        require('autoprefixer')({
            overrideBrowserslist: [
                "Android 4.1",
                "iOS 7.1",
                "Chrome > 31",
                "ff > 31",
                "ie >= 8"
                //'last 10 versions', // 所有主流浏览器最近10版本用
            ],
            grid: true
        })
    ]
}
  1. 打开 webpack.config.js 文件，找到 module -> rules 节点，配置 less 如下代码：
module: {
    rules: [
+       {
            test: /.less$/,
            use: [
                MiniCssExtractPlugin.loader,
                'css-loader',
                'less-loader',
                'postcss-loader'
            ],
        },
    ]
}

# px2rem-loader

移动端 CSS 可以 使⽤ px2rem-loader 将 px ⾃动转换成 rem，然后再利用 lib-flexible 在⻚⾯渲染时根据当前设备的宽高动态计算根元素的 font-size 值

  1. 安装
npm install px2rem-loader -D
npm install lib-flexible -S
  1. 打开 webpack.config.js 文件，找到 module -> rules 节点，配置 less 如下代码：
module: {
    rules: [
+       {
            test: /.less$/,
            use: [
                MiniCssExtractPlugin.loader,
                'css-loader',
                'less-loader',
                'postcss-loader',
                {
                    loader: 'px2rem-loader',
                    options: {
                        remUnit: 75,
                        remPrecision: 8,
                    },
                }
            ],
        },
    ]
}
  1. 配置好 px2rem-loader 之后就需要把 lib-flexible 内联到 html 中

那么要如何实现 JS 的内联呢？这里就需要用到 资源内联 的相关知识

# raw-loader

资源内联 就是如何把 css 或者 js 代码内联到 html 中，或者把图片、文字内联到代码中，其主要意义可以从两个方面探讨：

  • 代码层⾯：
    • ⻚⾯框架的初始化脚本
    • 上报点的相关代码也需要内联到html中
    • 首屏的 css 一般内联到 html 中，这样 html 加载回来的时候，css也完成了，可以避免⻚⾯闪动
  • 请求层⾯：
    • ⼩图⽚或者字体资源的内联可以减少 HTTP ⽹络请求数

raw-loader 可以实现 HTML 和 JS 的内联

raw-loader 通过读取一个文件的内容，返回一个 string，把 string 插入相应的位置，具体的使用方式如下：

  1. 安装
npm install raw-loader@0.5.1 -D
  1. 以上面将 lib-flexible 内联到 html 中为例子，需要在入口文件 index.html 的头部添加以下内容：
<!DOCTYPE html>
<html lang="en">
<head>
    <!-- raw-loader 内联 JS -->
    <script>${ require('raw-loader!babel-loader!../../node_modules/lib-flexible/flexible.js')}</script>
</head>
...
  • 其中 raw-loader!babel-loader! 后面是要内联的 JS 路径

这是内联 JS 的方法，raw-loader 同样可以内联 html，现在新建一个 meta.html 文件，并添加以下代码

<meta charset="UTF-8">
<meta name="viewport" content="viewport-fit=cover,width=device-width,initial-scale=1,user-scalable=no">
<meta name="format-detection" content="telephone=no">
<meta name="apple-mobile-web-app-capable" content="no">
<meta http-equiv="X-UA-Compatible" content="IE=Edge,chrome=1">
<link rel="dns-prefetch" href="//11.url.cn/">
<link rel="dns-prefetch" href="//open.mobile.qq.com/">

然后同样是在头部添加一些内容：

<head>
    <!-- raw-loader 内联 html -->
+   ${ require('raw-loader!./meta.html')}
</head>

这时候就可以执行构建命令就可以看到 meta.html 的内容已经被打包到入口 html 文件中

# html-inline-css-webpack-plugin

实现 CSS 的内联可以通过两种方式：

  • 借助 style-loader

    module.exports = {
    module: {
        rules: [
        {
            test: /\.scss$/,
            use: [
            {
                loader: 'style-loader',
                options: {
                    insertAt: 'top', // 样式插入到<head>
                    singleton: true, //将所有的style标签合并成一个
                }
            },
            "css-loader",
            "sass-loader"
            ],
        },
        ]
    },
};

  • html-inline-css-webpack-plugin

    1. 首先进行安装
    npm i html-inline-css-webpack-plugin -D
    1. 配置 plugins，并且注意 html-inline-css-webpack-plugin 需要放在 html-webpack-plugin 后面
      const HTMLInlineCSSWebpackPlugin = require("html-inline-css-webpack-plugin").default;

  module.exports = {
      plugins: [
          new MiniCssExtractPlugin({
              filename: '[name]_[contenthash:8].css'
          }),
          new HtmlWebpackPlugin(),
          new HTMLInlineCSSWebpackPlugin()
      ]
  }
