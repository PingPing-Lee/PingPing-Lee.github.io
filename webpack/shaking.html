# DCE

了解 tree shaking 之前先来了解一个名词 DCE(Dead code elimination)，解释起来就是把没有用到的代码 Dead code 擦除掉

这里提到的没有用到的代码可以具体指以下情况：

  • 代码不会被执⾏，不可到达
  • 代码执⾏的结果不会被⽤到
  • 代码只会影响死变量（只写不读）

# tree shaking

在 1 个模块中可能有多个⽅法，只要其中的某个⽅法使⽤到了，则整个⽂件都会被打到 bundle ⾥⾯去，tree shaking 也是沿用 DCE 的规则来分析哪些代码未被引用并标记，然后在 uglify 阶段把没⽤到的代码擦除掉，只把⽤到的⽅法打⼊ bundlejs 中

tree shaking 必须是 ES6 的语法，因为其主要利⽤ ES6 模块的特点:

  • 只能作为模块顶层的语句出现
  • import 的模块名只能是字符串常量
  • import binding 是 immutable的 代码擦除： uglify 阶段删除⽆⽤代码

在 webpack4 中默认⽀持，在打包时设置的 mode 为 production 的情况下默认开启，下面就来具体看看实现方式如下：

  1. 在我们的项目中添加一个新的通用模块文件 src/tree-shaking.js，并导出两个函数：
export function a() {
    return 'This is func a';
}
export function b() {
    return 'This is func b';
}
  1. 新建完成之后，更新入口 src/index.js 脚本，并将 src/tree-shaking.js 引入，但是并不使用其方法，具体代码如下：
// src/index.js
...
import { a } from './tree-shaking';

class Search extends React.Component {

    render() {
        return <div className="search-text">
            ...
        </div>;
    }
}
...
  • 这是其实 src/tree-shaking.js 中的 function a 和 function b 都是未被使用，就是所谓的“未引用代码(dead code)”

如果这时执行命令打包之后查看 bundle 文件发现，function a 和 function b 还是被打包进去了，如下图：

说明我们这个时候并没有真正的开启 tree shaking，想要解决这个问题还需要将 webpack.config.js 中 mode 选项设置为 production，从而启用 minification(代码压缩) 和 tree shaking，代码如下：

// webpack.config.js
module.exports = {
    ...
+   mode: 'production',
};

此时重新构建再次查看 bundle 文件，就不会存在 function a 和 function b，说明已经正确开启 tree shaking 了

# sideEffects

正常情况构建时会对所有模块进行 tree shaking，如果想要排除某些文件，需要在 package.json 中配置 sideEffects 属性，具体代码如下：

{
  "name": "your-project",
  "sideEffects": ["*.css"], // 表示除 .css 文件以外的进行 tree shaking  
}
