在日常项目中，不同页面可能会引入一些相同的基础库或者共用一些公共模块，在打包时会对公共资源进行重复打包，会造成打包体积过大，所以需要对一些公共资源的提取，比如基础库的分离、公共模块分离

# html-webpack-externals-plugin

在 react 或者 vue 开发中，我们需要引入基础库，以 react 为例，在很多页面都会引入 react、react-dom 等基础库，为了避免重复打包，我们可以借助html-webpack-externals-plugin 将 react、react-dom 通过 cdn 引⼊，不打⼊ bundle 中，具体如下图：

  1. 安装
npm install --save-dev html-webpack-externals-plugin
  1. 在 webpack.config.js 中配置 plugins，具体代码如下：
const HtmlWebpackExternalsPlugin = require('html-webpack-externals-plugin');
module.exports = {
    plugins: [
        ...
        new HtmlWebpackExternalsPlugin({
            externals: [
                {
                    module: 'react',
                    entry: 'https://now8.gtimg.com/now/lib/16.8.6/react.min.js?_bid=4042',
                    global: 'React',
                },
                {
                    module: 'react-dom',
                    entry: 'https://now8.gtimg.com/now/lib/16.8.6/react-dom.min.js?_bid=4042',
                    global: 'ReactDOM',
                },
            ],
        }),
    ]
};
  1. 最后记得在模版 html 中，新增 react、react-dom 脚本，如下：
...
<body>
    <div id="root"></div>
    <script type="text/javascript" src="https://now8.gtimg.com/now/lib/16.8.6/react.min.js?_bid=4042"></script>
    <script type="text/javascript" src="https://now8.gtimg.com/now/lib/16.8.6/react-dom.min.js?_bid=4042"></script>
</body>

其实除了这种方式以外，还可以利⽤ SplitChunksPlugin 进⾏公共脚本分离，同时 SplitChunksPlugin 也是常用的代码分离方法之一

# 代码分割

在日常的开发中，如果当前页面业务逻辑代码体积就不小，在引入第三方的工具库，这样就会导致如下问题：

  • 体积大，加载时间长 业务逻辑会变化
  • 第三方工具库不会，所以业务逻辑一变更，第三方工具库也要跟着变

所以此时最好的解决办法就是 代码分割

代码分割 是 webpack 中最引人注目的特性之一，此特性能够把代码分割成 chunks（语块），然后可以按需加载或并行加载这些文件，获取更小的 bundle，以及控制资源加载优先级，如果使用合理，会极大影响加载时间

适⽤的场景：

  • 抽离相同代码到⼀个共享块
  • 脚本懒加载，用到才会加载，初始化并不会加载，使得初始下载的代码更⼩

我们继续来看一下两种常用的代码分割方法：

  • 防止重复：使用 SplitChunksPlugin 去重和分割 chunk
  • 动态导入：通过模块的内联函数调用来分离代码

# SplitChunksPlugin

SplitChunksPlugin 是 Webpack4 功能非常强大的内置插件，用来替代之前的 CommonsChunkPlugin 插件，可以将公共的依赖模块提取到已有的 entry chunk 中，或者提取到一个新生成的 chunk，官网参考 (opens new window)

使用起来可配置很多参数，默认参数配置的基础代码示例如下：

module.exports = {
    optimization: {
        splitChunks: {
            chunks: 'async',
            minSize: 30000, // 抽离的公共包的最小体积
            maxSize: 0, // 最大的体积，单位是字节
            minChunks: 1, // 设置最小引用次数，大于设置的值就会被提取成公共文件
            maxAsyncRequests: 5,
            maxInitialRequests: 3, // 浏览器同时进行异步请求的资源的最大数
            automaticNameDelimiter: '~',
            name: true,
            cacheGroups: {
                defaultVendors: {
                    test: /[\\/]node_modules[\\/]/,
                    priority: -10,
                    reuseExistingChunk: true,
                },
                default: {
                    minChunks: 2,
                    priority: -20,
                    reuseExistingChunk: true,
                },
            }
        }
    }
}
  • chunks 参数说明：
    • async：默认值，对异步引⼊的库进⾏分离
    • initial：只对同步引⼊的库进⾏分离
    • all：对所有方式引⼊的库都进⾏分离(推荐)

想要分离 react、react-dom 基础包，代码如下：

module.exports = {
    optimization: {
        splitChunks: {
            cacheGroups: {
                commons: {
                    test: /(react|react-dom)/,
                    name: 'vendors',
                    chunks: 'all'
                }
            }
        }
    }
}

# 动态导入

动态 import 的意思是指不会在初始化的时间加载模块，使用到的时候才会加载，类似于 commonJS 中 require 一样按需加载

⽬前还没有原⽣⽀持 动态 import，所以需要 babel 转换，具体的使用方法如下：

  1. 安装 babel 插件
npm install @babel/plugin-syntax-dynamic-import --save-dev
  1. 把上面安装的 babel 插件添加到 .babelrc 的配置文件中：
{
    "presets": [
        "@babel/preset-env",
        "@babel/preset-react"
    ],
+   "plugins": ["@babel/plugin-syntax-dynamic-import"]
}
  1. 在项目的入口js 文件中，动态 import，如下：
'use strict';
import React from 'react';
import ReactDOM from 'react-dom';
import avatar from './images/avatar.png';

class Search extends React.Component {
    constructor(){
        super();
        this.state = {
            Text: null,
        }
    }
+    imgClick = () => {
+        import('./text.js').then((Text) => {
+            this.setState({
+                Text: Text.default
+            });
+        });
+    }
    render() {
        const { Text } = this.state;
        return <div className="search-text">
            {
                Text ? <Text /> : null
            }
            搜索文字的内容,改变试试
+           <img src={ avatar } onClick={this.imgClick}/>
        </div>;
    }
}
ReactDOM.render(
    <Search />,
    document.getElementById('root')
);
  • 给 img 标签添加点击事件
  • 在点击 img 的时候动态 import 另一个模块 text.js
  • import() 会返回一个 promise，所以可以用链式调用，在 then 中处理结果
