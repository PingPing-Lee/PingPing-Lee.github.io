服务端渲染（SSR） 是 Server Side Render 简称，与之相对的还有 客户端渲染（CSR） 是 Client Side Render 的简称

# CSR

了解 SSR 之前先来看一下客户端渲染（CSR） ，通过下图了解 浏览器渲染的整体流程

整个渲染过程简单点描述就是浏览器将 HTML 文本转化为页面帧的过程，需要浏览器先加载 HTML 构建 DOM 树，然后请求 CSS，与 CSS 解析生成的 CSS 规则树结合形成渲染树，加载JS 脚本数据，以及加载图片资源，计算布局并绘制图像，我们页面的初次渲染就大功告成了

# SSR

在浏览器请求页面URL的时候，服务端将 HTML、CSS、JS、Data进过一些的组装成 HTML文本，并返回给浏览器，这个HTML文本被浏览器解析之后，不需要经过 JavaScript 脚本的执行，即可直接构建出希望的 DOM 树并展示到页面中

这个服务端组装HTML的过程，叫做服务端渲染，大致流程图如下：

有图可以总结特点如下：

  • 所有模板等资源都存储在服务端
  • 内⽹机器拉取数据更快
  • ⼀个 HTML 返回所有数据
  • 服务端渲染 (SSR) 的核⼼是减少请求

对比：

客户端渲染 服务端渲染
请求 多个请求(HTML、数据等) 1个请求
加载过程 HTML&数据串行加载 1个请求返回HTML&数据
好处 网络传输数据量小、减少了服务器压力、前后端分离、局部刷新，无需每次请求完整页面、交互好可实现各种效果 首屏渲染快、利于SEO、可以生成缓存片段，生成静态化文件、节能（对比客户端渲染的耗电）
弊端 不利于SEO、爬虫看不到完整的程序源码、首屏渲染慢（渲染前需要下载一堆js和css等） 用户体验较差、不容易维护，通常前端改了部分html或者css，后端也需要修改
可交互 图片等静态资源加载完成，js 逻辑执行完成可交互

# 实现步骤

想要通过 Webpack 实现 SSR，服务端和客户端都需要有所改动，同时和常规的 CSR 代码书写也有不一样的点，下面用 webpack + React 框架新建一个项目并完成服务端渲染，详细的描述整体实现步骤：

  1. 初始化项目包，整体项目结构树如下：
|____server
| |____index.js
|____src
| |____index.html
| |____index.js
| |____index.less
|____.babelrc
|____package.json
|____webpack.ssr.js
  1. 服务端

在 server -> index.js 中配置内容如下：

  // 服务端没有 window 对象，需要处理一下
  if (typeof window === 'undefined') {
      global.window = {};
  }
  // 采用 express 启动 node 服务器，记得安装 npm i express -D
  const express = require('express');
  // react 组件中包含很多浏览器对象，如 document，window等，但是服务端是没有这些对象的，所以不能直接使用 ReactDOM 来渲染组件，而是使⽤ react-dom/server
  const { renderToString } = require('react-dom/server');
  // 引入构建之后的组件
  const SSR = require('../dist/index-server');
  const server = (port) => {
      const app = express();
      app.use(express.static('dist'));
      app.get('/index', (req,res) => {
          // 使⽤ react-dom/server 的 renderToString ⽅法将 React 组件渲染成字符串，然后丢到 HTML 模版中
          const html = renderMarkup(renderToString(SSR));
          res.status(200).send(html);
      });
      app.listen(port, () => {
          console.log('Server is running on port:' + port);
      });
  }
  server(process.env.PORT || 3000);
  const renderMarkup = (str) => {
      return `
      <!DOCTYPE html>
      <html lang="en">
      <head>
          <meta charset="UTF-8">
          <meta http-equiv="X-UA-Compatible" content="IE=edge">
          <meta name="viewport" content="width=device-width, initial-scale=1.0">
          <title>Document</title>
      </head>
      <body>
          <div id="root">${str}</div>
      </body>
      </html>
      `;
  }

  1. 配置 webpack.ssr.js
const path = require('path');
const HtmlWebpackPlugin = require('html-webpack-plugin');
const MiniCssExtractPlugin = require('mini-css-extract-plugin');
module.exports = {
    entry: './src/index.js',
    output: {
        path: path.join(__dirname, 'dist'),
        filename: 'index-server.js',
        libraryTarget: 'umd',
    },
    mode: 'none',
    module: {
        rules: [
            {test: /.js$/, use: ['babel-loader']},
            {test: /.less$/, use: [MiniCssExtractPlugin.loader, 'css-loader', 'less-loader']}
        ],
    },
    plugins: [
        new HtmlWebpackPlugin({
            title: '测试自定义标题',
            template: path.join(__dirname, 'src/index.html'), // 指定要用到的模版文件
            filename: 'index.html'    //指定生成的文件的名称，该文件存在于内存中，在目录中不显示
        }),
        new MiniCssExtractPlugin({
            filename: '[name]_[contenthash:8].css',
        })
    ],
};

此时在终端通过 npm run build:ssr 命令进行构建，然后通过 node server/index.js 启动服务器，在浏览器输入 http://127.0.0.1:3000/index 查看页面会发现页面样式并没有生效，因为 Node.js ⽆法解析 CSS

解决样式不显示的问题，可以有以下方案：

  • ⽅案⼀：服务端打包通过 ignore-loader 忽略掉 CSS 的解析

  • ⽅案⼆：将 style-loader 替换成 isomorphic-style-loader

  • 方案三：设置占位符，动态插⼊组件

    • 修改 src/index.html，添加占位符，如下：
     ...
 <body>
 -   <div id="root"></div>
 +   <div id="root"><!--HTML_PLACEHOLDER--></div>
 </body>
 ...
    • 修改 sercer/index.js 中模版字符的格式：
    const fs = require('fs');
const path = require('path');
// 使⽤打包出来的浏览器端 html 为模板
const template = fs.readFileSync(path.join(__dirname, '../dist/search.html'), 'utf-8');
...
const renderMarkup = (str) => {
  return template.replace('<!--HTML_PLACEHOLDER-->', str);
}

上述方案解决了样式问题，以此类推，我们在处理首屏数据时可以用服务端获取的数据替换占位符，具体操作如下：

  • 修改 src/index.html，添加占位符，如下：

     ...
 <body>
 +       <!--INITIAL_DATA_PLACEHOLDER-->
 </body>
 ...
    • 修改 sercer/index.js 中模版字符的格式：
    ...
const renderMarkup = (str) => {
  // 这里的 data 就是服务端获取的数据
  const dataStr = JSON.stringify(data);
  return template.replace('<!--HTML_PLACEHOLDER-->', str).replace('<!--INITIAL_DATA_PLACEHOLDER-->', `<script>window.__initial_data=${dataStr}</script>`);
}
